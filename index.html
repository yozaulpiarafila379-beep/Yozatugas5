1. Pengenalan Sealed Class

Sealed class adalah jenis class dalam Dart yang digunakan untuk membatasi pewarisan (inheritance), sehingga class tersebut hanya dapat diturunkan (di-extend) oleh class lain yang berada dalam file yang sama. Dengan adanya pembatasan ini, pengembang dapat mengontrol dengan jelas struktur turunan class, sehingga program menjadi lebih aman, terorganisir, dan mudah dipahami. Pada kode berikut, ApiResponse digunakan sebagai sealed class yang memiliki dua turunan, yaitu Success dan Error. Kedua class tersebut merepresentasikan kondisi berhasil dan gagal, sehingga memudahkan dalam pengelolaan alur program dan pengecekan tipe data.

Kode Program VS Code:


// Nama  : Yoza Ulpia Rafila
// NIM   : 251410004
// Kelas : SI2A

sealed class ApiResponse {}

class Success extends ApiResponse {
  final String message;
  Success(this.message);
}

class Error extends ApiResponse {
  final String errorMessage;
  Error(this.errorMessage);
}

void main() {
  print("Contoh penggunaan Sealed Class");
}

2. Keuntungan Penggunaan Sealed class

Keuntungan sealed class dalam Dart adalah dapat membatasi pewarisan hanya dalam satu file, sehingga semua subclass harus dibuat dalam file yang sama. Hal ini membuat program lebih terkontrol, aman, dan mudah dipahami karena semua kemungkinan turunan sudah diketahui. Pada kode berikut, ApiResponse digunakan sebagai sealed class dengan dua turunan, yaitu Success dan Error, yang mewakili kondisi berhasil dan gagal. Dengan adanya fungsi handleResponse(), program dapat melakukan pengecekan tipe untuk menampilkan hasil yang sesuai, sehingga alur program menjadi lebih rapi, jelas, dan terstruktur.

Kode Program VS Code:


// Nama  : Yoza Ulpia Rafila
// NIM   : 251410004
// Kelas : SI2A

sealed class ApiResponse {}

class Success extends ApiResponse {
  final String message;
  Success(this.message);
}

class Error extends ApiResponse {
  final String errorMessage;
  Error(this.errorMessage);
}

void handleResponse(ApiResponse response) {
  if (response is Success) {
    print("Success: ${response.message}");
  } else if (response is Error) {
    print("Error: ${response.errorMessage}");
  }
}

void main() {
  handleResponse(Success("Data berhasil diambil"));
  handleResponse(Error("Terjadi kesalahan"));
}

3. Polimorfisme (Method Overriding)

Polimorfisme adalah konsep dalam pemrograman berorientasi objek yang memungkinkan sebuah method memiliki banyak bentuk perilaku. Salah satu bentuk polimorfisme adalah method overriding, yaitu ketika subclass mendefinisikan ulang method yang sudah ada pada superclass. Pada kode berikut, class Animal memiliki method speak() yang menampilkan suara umum. Kemudian class Dog sebagai turunan dari Animal melakukan overriding pada method tersebut untuk menghasilkan suara yang berbeda, yaitu "Woof!". Hal ini menunjukkan bahwa method yang sama dapat memiliki perilaku yang berbeda tergantung pada objek yang memanggilnya.

Kode Program VS Code:


// Nama  : Yoza Ulpia Rafila
// NIM   : 251410004
// Kelas : SI2A

class Animal {
  void speak() {
    print("Hewan bersuara");
  }
}

class Dog extends Animal {
  @override
  void speak() {
    print("Anjing: Woof!");
  }
}

void main() {
  Animal a = Animal();
  a.speak();

  Dog d = Dog();
  d.speak();
}

4. Method Overriding Pada Class Turunan

Method overriding adalah konsep dalam pemrograman berorientasi objek di mana class turunan (subclass) mendefinisikan ulang method yang sudah ada pada class induk (superclass). Tujuannya adalah untuk memberikan perilaku yang berbeda sesuai dengan kebutuhan masing-masing objek. Pada kode berikut, class Animal memiliki method speak() yang menampilkan suara umum. Kemudian class Dog dan Cat sebagai turunan melakukan overriding terhadap method tersebut untuk menghasilkan suara yang berbeda, yaitu "Woof!" dan "Meow!". Hal ini menunjukkan bahwa method yang sama dapat memiliki perilaku yang berbeda tergantung pada objek yang digunakan. Dengan demikian, method overriding membantu membuat program lebih fleksibel, mudah dikembangkan, dan mendukung konsep polimorfisme dalam pemrograman.

Kode Program VS Code:


// Nama  : Yoza Ulpia Rafila
// NIM   : 251410004
// Kelas : SI2A

class Animal {
  void speak() {
    print("Hewan bersuara");
  }
}

class Dog extends Animal {
  @override
  void speak() {
    print("Dog: Woof!");
  }
}

class Cat extends Animal {
  @override
  void speak() {
    print("Cat: Meow!");
  }
}

void main() {
  Animal a = Animal();
  a.speak();

  Animal d = Dog();
  d.speak();

  Animal c = Cat();
  c.speak();
}

5. Final Class Settings dalam Dart

Final class adalah jenis class dalam Dart yang tidak dapat diwarisi (tidak bisa di-extend) oleh class lain. Hal ini bertujuan untuk menjaga agar struktur dan perilaku class tetap konsisten serta tidak dapat diubah melalui pewarisan. Pada kode berikut, class Settings didefinisikan sebagai final class yang memiliki atribut theme dan fontSize. Class ini digunakan untuk menyimpan dan menampilkan pengaturan aplikasi. Karena bersifat final, class Settings tidak dapat dijadikan sebagai superclass, sehingga lebih aman dari perubahan yang tidak diinginkan. Dengan demikian, penggunaan final class membantu menjaga kestabilan program dan memastikan bahwa class tersebut hanya digunakan sesuai dengan tujuan awal pembuatannya.

Kode Program VS Code:


// Nama  : Yoza Ulpia Rafila
// NIM   : 251410004
// Kelas : SI2A

final class Settings {
  String theme;
  int fontSize;

  Settings({required this.theme, required this.fontSize});

  void showSettings() {
    print("Theme: $theme, Font Size: $fontSize");
  }
}

void main() {
  Settings settings = Settings(theme: "Dark", fontSize: 14);
  settings.showSettings();
}

6. Pewarisan dan Method Overriding Pada Class Repository

Pewarisan (inheritance) adalah konsep dalam pemrograman berorientasi objek yang memungkinkan sebuah class turunan mewarisi atribut dan method dari class induk. Selain itu, class turunan juga dapat melakukan method overriding, yaitu mendefinisikan ulang method dari class induk sesuai kebutuhan. Pada kode berikut, class Repository merupakan class induk yang memiliki method create, read, update, dan delete. Kemudian class UserRepository sebagai turunan mewarisi semua method tersebut, namun melakukan overriding pada method read() untuk menampilkan data yang lebih spesifik, yaitu data user. Dengan demikian, penggunaan inheritance dan method overriding membuat program lebih fleksibel, mengurangi duplikasi kode, serta memudahkan pengembangan sesuai kebutuhan.

Kode Program VS Code:


// Nama  : Yoza Ulpia Rafila
// NIM   : 251410004
// Kelas : SI2A

class Repository {
  void create() {
    print("Data berhasil ditambahkan");
  }

  void read() {
    print("Menampilkan data");
  }

  void update() {
    print("Data berhasil diupdate");
  }

  void delete() {
    print("Data berhasil dihapus");
  }
}

class UserRepository extends Repository {
  @override
  void read() {
    print("Menampilkan data user");
  }
}

void main() {
  UserRepository repo = UserRepository();

  repo.create();
  repo.read();
  repo.update();
  repo.delete();
}

7. Penggunaan Ultility Class dengan Static Method

Utility class adalah class yang digunakan untuk menyediakan kumpulan method yang bersifat umum dan dapat digunakan tanpa harus membuat objek. Biasanya, utility class berisi method static yang dapat dipanggil langsung melalui nama class. Pada kode berikut, class MathUtils merupakan utility class yang didefinisikan sebagai final class dan memiliki method static yaitu tambah() dan kali(). Method tersebut digunakan untuk melakukan operasi matematika sederhana tanpa perlu membuat objek dari class MathUtils. Dengan demikian, penggunaan utility class dengan static method membuat kode menjadi lebih efisien, praktis, dan mudah digunakan kembali dalam berbagai bagian program.

Kode Program VS Code:


// Nama  : Yoza Ulpia Rafila
// NIM   : 251410004
// Kelas : SI2A

final class MathUtils {
  static int tambah(int a, int b) {
    return a + b;
  }

  static int kali(int a, int b) {
    return a * b;
  }
}

void main() {
  print("Penjumlahan: ${MathUtils.tambah(5, 3)}");
  print("Perkalian: ${MathUtils.kali(5, 3)}");
}
