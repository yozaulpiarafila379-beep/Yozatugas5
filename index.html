1.Sealed Class dalam Dart

Pengertian Sealed Class:

  • Sealed class adalah jenis class dalam Dart yang membatasi pewarisan hanya dalam satu file yang sama.
  • Pembatasan ini memungkinkan pengembang untuk mengontrol semua class turunan secara lebih jelas.
  • Dengan sealed class, program menjadi lebih aman, terorganisir, dan mudah dipahami.
  • Pada program, ApiResponse digunakan sebagai sealed class dengan dua turunan, yaitu Success dan Error.
  • Kedua class tersebut merepresentasikan kondisi berhasil dan gagal.
  • Hal ini memudahkan pengelolaan alur program serta pengecekan tipe data.

Kode Program VS Code:


// Nama  : Yoza Ulpia Rafila
// NIM   : 251410004
// Kelas : SI2A

sealed class ApiResponse {}

class Success extends ApiResponse {
  final String message;
  Success(this.message);
}

class Error extends ApiResponse {
  final String errorMessage;
  Error(this.errorMessage);
}

void main() {
  print("Contoh penggunaan Sealed Class");
}

2.Keuntungan Penggunaan Sealed Class

Pengertian Keuntungan penggunaan Sealed Class:

  • Sealed class adalah class dalam Dart yang membatasi pewarisan hanya dalam satu file.
  • Semua subclass harus dibuat dalam file yang sama sehingga lebih terkontrol.
  • Hal ini membuat program lebih aman dan mudah dipahami karena semua kemungkinan turunan sudah diketahui.
  • Pada program, ApiResponse digunakan sebagai sealed class dengan dua turunan, yaitu Success dan Error.
  • Kedua class tersebut mewakili kondisi berhasil dan gagal.
  • Fungsi handleResponse() digunakan untuk mengecek tipe dan menampilkan hasil yang sesuai.
  • Penggunaan sealed class membuat alur program lebih rapi, jelas, dan terstruktur.

Kode Program VS Code:


// Nama  : Yoza Ulpia Rafila
// NIM   : 251410004
// Kelas : SI2A

sealed class ApiResponse {}

class Success extends ApiResponse {
  final String message;
  Success(this.message);
}

class Error extends ApiResponse {
  final String errorMessage;
  Error(this.errorMessage);
}

void handleResponse(ApiResponse response) {
  if (response is Success) {
    print("Success: ${response.message}");
  } else if (response is Error) {
    print("Error: ${response.errorMessage}");
  }
}

void main() {
  handleResponse(Success("Data berhasil diambil"));
  handleResponse(Error("Terjadi kesalahan"));
}

3.Polimorfisme dalam OOP (Method Overriding)

Pengertian Polimorfisme (Method Overriding):

  • Polimorfisme adalah konsep dalam OOP yang memungkinkan sebuah method memiliki banyak bentuk perilaku.
  • Salah satu bentuk polimorfisme adalah method overriding, yaitu ketika subclass mendefinisikan ulang method dari superclass.
  • Pada program, class Animal memiliki method speak() sebagai method umum.
  • Class Dog sebagai turunan melakukan overriding pada method tersebut untuk menghasilkan suara "Woof!".
  • Hal ini menunjukkan bahwa method yang sama dapat memiliki perilaku yang berbeda tergantung pada objek yang memanggilnya.
  • Polimorfisme membuat program lebih fleksibel, mudah dikembangkan, dan dapat digunakan kembali.

Kode Program VS Code:


// Nama  : Yoza Ulpia Rafila
// NIM   : 251410004
// Kelas : SI2A

class Animal {
  void speak() {
    print("Hewan bersuara");
  }
}

class Dog extends Animal {
  @override
  void speak() {
    print("Anjing: Woof!");
  }
}

void main() {
  Animal a = Animal();
  a.speak();

  Dog d = Dog();
  d.speak();
}

4.Method Overriding pada Class Turunan

Pengertian Method Overriding:

  • Method overriding adalah konsep dalam OOP di mana class turunan mendefinisikan ulang method yang sudah ada pada class induk.
  • Tujuannya adalah untuk memberikan perilaku yang berbeda sesuai dengan kebutuhan masing-masing objek.
  • Pada program, class Animal memiliki method speak() sebagai method umum.
  • Class Dog dan Cat sebagai turunan melakukan overriding pada method tersebut untuk menghasilkan suara yang berbeda, yaitu "Woof!" dan "Meow!".
  • Hal ini menunjukkan bahwa satu method dapat memiliki banyak perilaku tergantung pada objek yang digunakan.
  • Method overriding mendukung konsep polimorfisme serta membuat program lebih fleksibel dan mudah dikembangkan.

5.Final Class dalam Dart

Pengertian Final Class:

  • Final class adalah class dalam Dart yang tidak dapat diwarisi atau di-extend oleh class lain.
  • Tujuan penggunaan final class adalah untuk menjaga konsistensi struktur dan perilaku class agar tidak berubah.
  • Pada program, class Settings merupakan final class yang memiliki atribut seperti theme dan fontSize.
  • Class ini digunakan untuk menyimpan dan mengelola pengaturan aplikasi.
  • Karena tidak dapat diwarisi, class ini lebih aman dari perubahan yang tidak diinginkan melalui pewarisan.
  • Penggunaan final class membantu menjaga kestabilan dan keamanan program.

Kode Program VS Code:


// Nama  : Yoza Ulpia Rafila
// NIM   : 251410004
// Kelas : SI2A

final class Settings {
  String theme;
  int fontSize;

  Settings({required this.theme, required this.fontSize});

  void showSettings() {
    print("Theme: $theme, Font Size: $fontSize");
  }
}

void main() {
  Settings settings = Settings(theme: "Dark", fontSize: 14);
  settings.showSettings();
}

6.Konsep Inheritance dalam OOP

( Bonus 1 )

Pengertian Pewarisan (Inheritance):

  • Pewarisan (inheritance) adalah konsep dalam OOP yang memungkinkan class turunan mewarisi atribut dan method dari class induk.
  • Konsep ini membantu mengurangi penulisan kode yang berulang (duplikasi).
  • Class turunan dapat melakukan method overriding, yaitu mendefinisikan ulang method dari class induk sesuai kebutuhan.
  • Pada program, class Repository sebagai class induk memiliki method create, read, update, dan delete.
  • Class UserRepository sebagai turunan mewarisi method tersebut dan melakukan overriding pada method read() untuk menampilkan data user.
  • Penggunaan inheritance membuat program lebih fleksibel, efisien, dan mudah dikembangkan.

Kode Program VS Code:


// Nama  : Yoza Ulpia Rafila
// NIM   : 251410004
// Kelas : SI2A

class Repository {
  void create() {
    print("Data berhasil ditambahkan");
  }

  void read() {
    print("Menampilkan data");
  }

  void update() {
    print("Data berhasil diupdate");
  }

  void delete() {
    print("Data berhasil dihapus");
  }
}

class UserRepository extends Repository {
  @override
  void read() {
    print("Menampilkan data user");
  }
}

void main() {
  UserRepository repo = UserRepository();

  repo.create();
  repo.read();
  repo.update();
  repo.delete();
}

7.Penggunaan Utility Class dalam Dart

( Bonus 2 )

Pengertian Utility Class:

  • Utility class adalah class yang digunakan untuk menyediakan kumpulan method yang bersifat umum dan dapat digunakan tanpa harus membuat objek.
  • Biasanya, utility class berisi method static yang dapat dipanggil langsung melalui nama class.
  • Pada program, class MathUtils merupakan utility class yang didefinisikan sebagai final class dan memiliki method static yaitu tambah() dan kali().
  • Method tersebut digunakan untuk melakukan operasi matematika sederhana tanpa perlu membuat objek dari class MathUtils.
  • Penggunaan utility class membuat kode menjadi lebih efisien, praktis, dan mudah digunakan kembali.
  • Class ini tidak dapat diinstansiasi karena hanya menggunakan static method.

Kode Program VS Code:


// Nama  : Yoza Ulpia Rafila
// NIM   : 251410004
// Kelas : SI2A

final class MathUtils {
  static int tambah(int a, int b) {
    return a + b;
  }

  static int kali(int a, int b) {
    return a * b;
  }
}

void main() {
  print("Penjumlahan: ${MathUtils.tambah(5, 3)}");
  print("Perkalian: ${MathUtils.kali(5, 3)}");
}

8.Konsep Abstraksi dalam OOP

Pengertian Penerapan OOP Abstraksi pada Dart:

  • Penerapan OOP: Abstraksi pada Dart merupakan penggunaan salah satu konsep Object-Oriented Programming (OOP) untuk menyederhanakan program dengan menyembunyikan detail implementasi dan hanya menampilkan fungsi utama.
  • Abstraksi membantu programmer dalam membuat program yang lebih terstruktur, mudah dipahami, dan mudah dikembangkan.
  • Dalam bahasa Dart, abstraksi dapat diterapkan menggunakan abstract class, yaitu class yang tidak dapat dibuat objek secara langsung.
  • Abstract class berisi method abstrak yang harus diimplementasikan oleh class turunan sesuai dengan kebutuhan masing-masing.
  • Pada contoh program, abstraksi diterapkan pada class Bentuk yang memiliki method hitungLuas(). Method tersebut kemudian diimplementasikan oleh class Persegi, Lingkaran, dan Segitiga untuk menghitung luas masing-masing bentuk dengan cara yang berbeda.
  • Abstract class tidak dapat dibuat objek secara langsung sehingga hanya digunakan sebagai kerangka dasar bagi class turunan.

Kode Dart:


// Nama  : Yoza Ulpia Rafila
// NIM   : 251410004
// Kelas : SI2A

// Class Abstrak
abstract class Bentuk {
  double hitungLuas();
}

// Class Persegi
class Persegi extends Bentuk {
  double sisi;

  Persegi(this.sisi);

  @override
  double hitungLuas() {
    return sisi * sisi;
  }
}

// Class Lingkaran
class Lingkaran extends Bentuk {
  double jariJari;

  Lingkaran(this.jariJari);

  @override
  double hitungLuas() {
    return 3.14 * jariJari * jariJari;
  }
}

// Class Segitiga
class Segitiga extends Bentuk {
  double alas;
  double tinggi;

  Segitiga(this.alas, this.tinggi);

  @override
  double hitungLuas() {
    return 0.5 * alas * tinggi;
  }
}

// Program Utama
void main() {
  // Objek Persegi
  Persegi persegi = Persegi(4);
  print("Luas Persegi: ${persegi.hitungLuas()}");

  // Objek Lingkaran
  Lingkaran lingkaran = Lingkaran(7);
  print("Luas Lingkaran: ${lingkaran.hitungLuas()}");

    
  // Objek Segitiga
  Segitiga segitiga = Segitiga(6, 8);
  print("Luas Segitiga: ${segitiga.hitungLuas()}");
}

10.Penggunaan Mixin dalam Dart

Pengertian Mixin dengan State pada Dart:

  • Penggunaan mixin dengan state pada Dart merupakan penerapan konsep Object-Oriented Programming (OOP) yang memungkinkan sebuah class menggunakan kembali method dan properti dari mixin tanpa harus menggunakan pewarisan secara langsung.
  • Mixin dengan state berarti mixin tidak hanya berisi method, tetapi juga memiliki properti (state) yang digunakan untuk menyimpan kondisi tertentu.
  • Dalam bahasa Dart, mixin digunakan dengan keyword with, sehingga sebuah class dapat memiliki lebih dari satu perilaku dari berbagai mixin.
  • Pada contoh program, mixin CanFly dan CanSwim memiliki state berupa isFlying dan isSwimming untuk mengatur kondisi objek.
  • Class seperti Bird, Duck, Fish, dan Airplane menggunakan mixin tersebut untuk mendapatkan kemampuan terbang dan berenang sesuai kebutuhan.
  • Dengan demikian, penggunaan mixin dengan state membuat program lebih fleksibel, efisien, dan mudah dikembangkan.

Kode Dart:


// Nama  : Yoza Ulpia Rafila
// NIM   : 251410004
// Kelas : SI2A

// Mixin dengan properti (state)
mixin CanFly {
  bool _isFlying = false;

  bool get isFlying => _isFlying;

  void fly() {
    if (!_isFlying) {
      _isFlying = true;
      print("Started flying!");
    } else {
      print("Already flying!");
    }
  }

  void land() {
    if (_isFlying) {
      _isFlying = false;
      print("Landed!");
    } else {
      print("Already on the ground!");
    }
  }
}

mixin CanSwim {
  bool _isSwimming = false;

  bool get isSwimming => _isSwimming;

  void swim() {
    if (!_isSwimming) {
      _isSwimming = true;
      print("Started swimming!");
    } else {
      print("Already swimming!");
    }
  }

  void stopSwim() {
    if (_isSwimming) {
      _isSwimming = false;
      print("Stopped swimming!");
    } else {
      print("Already stopped swimming!");
    }
  }
}

// Class dengan mixin (yang benar secara konsep OOP)
class Bird with CanFly {}

class Duck with CanFly, CanSwim {}

class Fish with CanSwim {}

class Airplane with CanFly {}

void main() {
  print("=== Bird ===");
  Bird bird = Bird();
  bird.fly();
  bird.land();

  print("\n=== Duck ===");
  Duck duck = Duck();
  duck.fly();
  duck.swim();
  duck.stopSwim();

  print("\n=== Fish ===");
  Fish fish = Fish();
  fish.swim();

  print("\n=== Airplane ===");
  Airplane airplane = Airplane();
  airplane.fly();
}

11.Sealed Class dan Pattern Matching dalam Dart

Pengertian Implementasi Sealed Class dengan Pattern Matching pada Dart:

  • Implementasi sealed class dengan pattern matching pada Dart merupakan penerapan konsep Object-Oriented Programming (OOP) yang digunakan untuk membatasi pewarisan class serta mempermudah pengelolaan berbagai kondisi dalam suatu program.
  • Sealed class memungkinkan pengembang untuk menentukan semua kemungkinan subclass dalam satu file, sehingga struktur program menjadi lebih terkontrol dan aman.
  • Pattern matching digunakan untuk memeriksa tipe objek dan langsung mengekstrak nilai yang ada di dalamnya dengan lebih ringkas dan jelas.
  • Dalam bahasa Dart, pattern matching biasanya digunakan dalam struktur switch-case untuk menangani berbagai kondisi berdasarkan tipe class.
  • Pada contoh program, class Status memiliki turunan seperti Loading, Loaded, dan Failed yang masing-masing merepresentasikan kondisi berbeda.
  • Dengan menggunakan pattern matching, program dapat menampilkan hasil yang sesuai dengan kondisi tersebut secara lebih terstruktur dan efisien.
  • Dengan demikian, penggunaan sealed class dan pattern matching membuat program lebih rapi, mudah dipahami, serta meminimalkan kesalahan dalam penanganan kondisi.

Kode Dart:


// Nama  : Yoza Ulpia Rafila
// NIM   : 251410004
// Kelas : SI2A

// Sealed class
sealed class Status {
  const Status();
}

class Loading extends Status {
  const Loading();
}

class Loaded extends Status {
  final String data;
  const Loaded(this.data);
}

class Failed extends Status {
  final String error;
  const Failed(this.error);
}

// Function handle status
void handleStatus(Status status) {
  switch (status) {
    case Loading():
      print("Loading...");
    case Loaded(:final data):
      print("Data: $data");
    case Failed(:final error):
      print("Error: $error");
  }
}

void main() {
  handleStatus(const Loading());
  handleStatus(const Loaded("Data berhasil"));
  handleStatus(const Failed("Terjadi error"));
}

12.State Management dengan Sealed Class dalam Dart

Pengertian Studi Kasus State Management Menggunakan Sealed Class pada Dart:

  • Studi kasus state management menggunakan sealed class pada Dart merupakan penerapan konsep Object-Oriented Programming (OOP) untuk mengelola berbagai kondisi (state) dalam suatu aplikasi secara terstruktur dan sistematis.
  • Sealed class digunakan untuk membatasi pewarisan sehingga semua kemungkinan state harus didefinisikan dalam satu file yang sama.
  • Hal ini memungkinkan pengembang untuk mengetahui seluruh kondisi yang mungkin terjadi, seperti loading, success, dan error.
  • Pada contoh program, class AppState digunakan sebagai sealed class dengan turunan LoadingState, SuccessState, dan ErrorState yang merepresentasikan kondisi berbeda dalam aplikasi.
  • Dengan menggunakan pattern matching pada struktur switch-case, program dapat menentukan tindakan yang sesuai berdasarkan jenis state yang diterima.
  • Pendekatan ini membuat alur program lebih jelas, terorganisir, serta mempermudah pengelolaan dan pengembangan aplikasi.
  • Dengan demikian, penggunaan sealed class dalam state management membantu meningkatkan kejelasan struktur, efisiensi, dan kualitas kode program.

Kode Dart:


// Nama  : Yoza Ulpia Rafila
// NIM   : 251410004
// Kelas : SI2A

// Sealed class
sealed class AppState {}

// State Loading
class LoadingState extends AppState {}

// State Success
class SuccessState extends AppState {
  final List items;
  SuccessState(this.items);
}

// State Error
class ErrorState extends AppState {
  final String message;
  ErrorState(this.message);
}

// Handler (Pattern Matching)
void updateUI(AppState state) {
  switch (state) {
    case LoadingState():
      showLoading();
    case SuccessState(:final items):
      showItems(items);
    case ErrorState(:final message):
      showError(message);
  }
}

// UI Functions
void showLoading() {
  print("⏳ Loading...");
}

void showItems(List items) {
  print("✅ Success! Items:");
  for (int i = 0; i < items.length; i++) {
    print("${i + 1}. ${items[i]}");
  }
}

void showError(String message) {
  print("❌ Error: $message");
}

// Main Program
void main() {
  updateUI(LoadingState());
  updateUI(SuccessState(["Item A", "Item B", "Item C"]));
  updateUI(ErrorState("Gagal memuat data"));
}
