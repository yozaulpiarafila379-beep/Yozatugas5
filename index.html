1.Sealed Class

  • Pada program ini, ApiResponse digunakan sebagai sealed class untuk membatasi pembuatan subclass.
  • Seluruh subclass yaitu Success dan Error dibuat dalam satu file yang sama.
  • Class Success merepresentasikan kondisi berhasil, sedangkan Error merepresentasikan kondisi gagal.
  • Dengan penggunaan sealed class, program hanya memiliki dua kondisi yang sudah ditentukan sejak awal.
  • Hal ini membuat alur program lebih terkontrol dan mudah dipahami.
  • Pada fungsi main(), program hanya menampilkan contoh penggunaan sealed class.

2.Keuntungan Sealed Class

Pembahasan:

  • Sealed class merupakan class dalam Dart yang membatasi pewarisan hanya dalam satu file.
  • Dengan pembatasan tersebut, semua subclass harus dibuat dalam file yang sama sehingga struktur program menjadi lebih terkontrol.
  • Hal ini membuat program lebih aman dan mudah dipahami karena seluruh kemungkinan turunan sudah diketahui sejak awal.
  • Pada program, ApiResponse digunakan sebagai sealed class dengan dua turunan, yaitu Success dan Error.
  • Kedua class tersebut mewakili kondisi berhasil dan gagal dalam program.
  • Fungsi handleResponse() digunakan untuk mengecek tipe data dan menampilkan hasil yang sesuai berdasarkan kondisi tersebut.
  • Dengan demikian, penggunaan sealed class membuat alur program lebih rapi, jelas, dan terstruktur.

Kode Program VS Code:


// Nama  : Yoza Ulpia Rafila
// NIM   : 251410004
// Kelas : SI2A

sealed class ApiResponse {}

class Success extends ApiResponse {
  final String message;
  Success(this.message);
}

class Error extends ApiResponse {
  final String errorMessage;
  Error(this.errorMessage);
}

void handleResponse(ApiResponse response) {
  if (response is Success) {
    print("Success: ${response.message}");
  } else if (response is Error) {
    print("Error: ${response.errorMessage}");
  }
}

void main() {
  handleResponse(Success("Data berhasil diambil"));
  handleResponse(Error("Terjadi kesalahan"));
}

3.Polimorfisme (Method Overriding)

Pembahasan Polimorfisme (Method Overriding):

  • Polimorfisme adalah konsep dalam OOP yang memungkinkan sebuah method memiliki banyak bentuk perilaku.
  • Salah satu bentuk polimorfisme adalah method overriding, yaitu ketika subclass mendefinisikan ulang method dari superclass.
  • Pada program, class Animal memiliki method speak() sebagai method dasar.
  • Class Dog sebagai turunan melakukan overriding pada method tersebut untuk menghasilkan suara "Woof!".
  • Hal ini menunjukkan bahwa method yang sama dapat memiliki perilaku yang berbeda tergantung pada objek yang memanggilnya.
  • Dengan demikian, polimorfisme membuat program lebih fleksibel, mudah dikembangkan, dan dapat digunakan kembali.

Kode Program VS Code:


// Nama  : Yoza Ulpia Rafila
// NIM   : 251410004
// Kelas : SI2A

class Animal {
  void speak() {
    print("Hewan bersuara");
  }
}

class Dog extends Animal {
  @override
  void speak() {
    print("Anjing: Woof!");
  }
}

void main() {
  Animal a = Animal();
  a.speak();

  Dog d = Dog();
  d.speak();
}

4.Method Overriding

Pembahasan Method Overriding:

  • Method overriding adalah konsep dalam OOP di mana class turunan mendefinisikan ulang method yang sudah ada pada class induk.
  • Tujuannya adalah untuk memberikan perilaku yang berbeda sesuai dengan kebutuhan masing-masing objek.
  • Pada program, class Animal memiliki method speak() sebagai method dasar.
  • Class Dog dan Cat sebagai turunan melakukan overriding pada method tersebut untuk menghasilkan suara yang berbeda, yaitu "Woof!" dan "Meow!".
  • Hal ini menunjukkan bahwa satu method dapat memiliki banyak perilaku tergantung pada objek yang digunakan.
  • Dengan demikian, method overriding mendukung konsep polimorfisme serta membuat program lebih fleksibel dan mudah dikembangkan.

Kode Program VS Code:


// Nama  : Yoza Ulpia Rafila
// NIM   : 251410004
// Kelas : SI2A

class Animal {
  void speak() {
    print("Hewan bersuara");
  }
}

class Dog extends Animal {
  @override
  void speak() {
    print("Dog: Woof!");
  }
}

class Cat extends Animal {
  @override
  void speak() {
    print("Cat: Meow!");
  }
}

void main() {
  Animal a = Animal();
  a.speak();

  Animal d = Dog();
  d.speak();

  Animal c = Cat();
  c.speak();
}

5.Final Class

Pembahasan Final Class:

  • Final class adalah class dalam Dart yang tidak dapat diwariskan (di-extend) oleh class lain.
  • Tujuan penggunaan final class adalah untuk menjaga konsistensi struktur dan perilaku class agar tidak berubah.
  • Pada program, class Settings merupakan final class yang memiliki atribut seperti theme dan fontSize.
  • Class ini digunakan untuk menyimpan dan mengelola pengaturan aplikasi.
  • Karena tidak dapat diwarisi, final class lebih aman dari perubahan yang tidak diinginkan melalui pewarisan.
  • Dengan demikian, penggunaan final class membantu menjaga stabilitas, keamanan, dan konsistensi program.

Kode Program VS Code:


// Nama  : Yoza Ulpia Rafila
// NIM   : 251410004
// Kelas : SI2A

final class Settings {
  String theme;
  int fontSize;

  Settings({required this.theme, required this.fontSize});

  void showSettings() {
    print("Theme: $theme, Font Size: $fontSize");
  }
}

void main() {
  Settings settings = Settings(theme: "Dark", fontSize: 14);
  settings.showSettings();
}

6.Inheritance

( Bonus 1 )

Pembahasan Pewarisan (Inheritance):

  • Pewarisan (inheritance) adalah konsep dalam OOP yang memungkinkan class turunan mewarisi atribut dan method dari class induk.
  • Konsep ini membantu mengurangi penulisan kode yang berulang (duplikasi).
  • Class turunan dapat melakukan method overriding, yaitu mendefinisikan ulang method dari class induk sesuai kebutuhan.
  • Pada program, class Repository sebagai class induk memiliki method create(), read(), update(), dan delete().
  • Class UserRepository sebagai turunan mewarisi method tersebut dan melakukan overriding pada method read() untuk menampilkan data user.
  • Dengan demikian, penggunaan inheritance membuat program lebih fleksibel, efisien, dan mudah dikembangkan.

Kode Program VS Code:


// Nama  : Yoza Ulpia Rafila
// NIM   : 251410004
// Kelas : SI2A

class Repository {
  void create() {
    print("Data berhasil ditambahkan");
  }

  void read() {
    print("Menampilkan data");
  }

  void update() {
    print("Data berhasil diupdate");
  }

  void delete() {
    print("Data berhasil dihapus");
  }
}

class UserRepository extends Repository {
  @override
  void read() {
    print("Menampilkan data user");
  }
}

void main() {
  UserRepository repo = UserRepository();

  repo.create();
  repo.read();
  repo.update();
  repo.delete();
}

7.Utility Class

( Bonus 2 )

Pembahasan Utility Class:

  • Utility class adalah class yang digunakan untuk menyediakan kumpulan method yang bersifat umum dan dapat digunakan tanpa harus membuat objek.
  • Biasanya, utility class berisi method static yang dapat dipanggil langsung melalui nama class.
  • Pada program, class MathUtils merupakan utility class yang didefinisikan sebagai final class dan memiliki method static yaitu tambah() dan kali().
  • Method tersebut digunakan untuk melakukan operasi matematika sederhana tanpa perlu membuat objek dari class MathUtils.
  • Penggunaan utility class membuat kode menjadi lebih efisien, praktis, dan mudah digunakan kembali.
  • Dengan demikian, utility class membantu menyediakan fungsi umum yang dapat diakses secara langsung tanpa proses instansiasi.

Kode Program VS Code:


// Nama  : Yoza Ulpia Rafila
// NIM   : 251410004
// Kelas : SI2A

final class MathUtils {
  static int tambah(int a, int b) {
    return a + b;
  }

  static int kali(int a, int b) {
    return a * b;
  }
}

void main() {
  print("Penjumlahan: ${MathUtils.tambah(5, 3)}");
  print("Perkalian: ${MathUtils.kali(5, 3)}");
}

8.Abstraksi

Pembahasan Penerapan OOP Abstraksi pada Dart:

  • Abstraksi merupakan salah satu konsep dalam Object-Oriented Programming (OOP) yang digunakan untuk menyederhanakan program dengan menyembunyikan detail implementasi dan hanya menampilkan fungsi utama.
  • Abstraksi membantu programmer dalam membuat program yang lebih terstruktur, mudah dipahami, dan mudah dikembangkan.
  • Dalam bahasa Dart, abstraksi dapat diterapkan menggunakan abstract class, yaitu class yang tidak dapat dibuat objek secara langsung.
  • Abstract class berisi method abstrak yang harus diimplementasikan oleh class turunan sesuai dengan kebutuhan masing-masing.
  • Pada contoh program, abstraksi diterapkan pada class Bentuk yang memiliki method hitungLuas(). Method tersebut kemudian diimplementasikan oleh class Persegi, Lingkaran, dan Segitiga untuk menghitung luas masing-masing bentuk dengan cara yang berbeda.
  • Dengan demikian, abstraksi berfungsi sebagai kerangka dasar yang membantu menyederhanakan struktur program serta memudahkan pengembangan.

Kode Dart:


// Nama  : Yoza Ulpia Rafila
// NIM   : 251410004
// Kelas : SI2A

// Class Abstrak
abstract class Bentuk {
  double hitungLuas();
}

// Class Persegi
class Persegi extends Bentuk {
  double sisi;

  Persegi(this.sisi);

  @override
  double hitungLuas() {
    return sisi * sisi;
  }
}

// Class Lingkaran
class Lingkaran extends Bentuk {
  double jariJari;

  Lingkaran(this.jariJari);

  @override
  double hitungLuas() {
    return 3.14 * jariJari * jariJari;
  }
}

// Class Segitiga
class Segitiga extends Bentuk {
  double alas;
  double tinggi;

  Segitiga(this.alas, this.tinggi);

  @override
  double hitungLuas() {
    return 0.5 * alas * tinggi;
  }
}

// Program Utama
void main() {
  // Objek Persegi
  Persegi persegi = Persegi(4);
  print("Luas Persegi: ${persegi.hitungLuas()}");

  // Objek Lingkaran
  Lingkaran lingkaran = Lingkaran(7);
  print("Luas Lingkaran: ${lingkaran.hitungLuas()}");

    
  // Objek Segitiga
  Segitiga segitiga = Segitiga(6, 8);
  print("Luas Segitiga: ${segitiga.hitungLuas()}");
}

  • Dengan abstraksi, program hanya menampilkan fungsi utama sehingga lebih mudah dipahami dan digunakan.
  • Dalam bahasa Dart, abstraksi diterapkan menggunakan abstract class, yaitu class yang tidak dapat dibuat objek secara langsung dan berfungsi sebagai kerangka dasar bagi class turunan.

10.Mixin

Pembahasan Penggunaan Mixin pada Dart:

  • Pada program, mixin digunakan untuk menambahkan perilaku ke dalam class tanpa harus menggunakan pewarisan secara langsung.
  • Mixin tidak hanya berisi method, tetapi juga memiliki properti (state) yang digunakan untuk menyimpan kondisi tertentu.
  • Penggunaan keyword with memungkinkan sebuah class memiliki lebih dari satu perilaku dari berbagai mixin.
  • Pada contoh program, mixin CanFly dan CanSwim memiliki state berupa isFlying dan isSwimming untuk mengatur kondisi objek.
  • Class seperti Bird, Duck, Fish, dan Airplane menggunakan mixin tersebut untuk mendapatkan kemampuan terbang dan berenang sesuai kebutuhan.
  • Dengan pendekatan ini, program menjadi lebih fleksibel, efisien, dan mudah dikembangkan.

Kode Dart:


// Nama  : Yoza Ulpia Rafila
// NIM   : 251410004
// Kelas : SI2A

// Mixin dengan properti (state)
mixin CanFly {
  bool _isFlying = false;

  bool get isFlying => _isFlying;

  void fly() {
    if (!_isFlying) {
      _isFlying = true;
      print("Started flying!");
    } else {
      print("Already flying!");
    }
  }

  void land() {
    if (_isFlying) {
      _isFlying = false;
      print("Landed!");
    } else {
      print("Already on the ground!");
    }
  }
}

mixin CanSwim {
  bool _isSwimming = false;

  bool get isSwimming => _isSwimming;

  void swim() {
    if (!_isSwimming) {
      _isSwimming = true;
      print("Started swimming!");
    } else {
      print("Already swimming!");
    }
  }

  void stopSwim() {
    if (_isSwimming) {
      _isSwimming = false;
      print("Stopped swimming!");
    } else {
      print("Already stopped swimming!");
    }
  }
}

// Class dengan mixin (yang benar secara konsep OOP)
class Bird with CanFly {}

class Duck with CanFly, CanSwim {}

class Fish with CanSwim {}

class Airplane with CanFly {}

void main() {
  print("=== Bird ===");
  Bird bird = Bird();
  bird.fly();
  bird.land();

  print("\n=== Duck ===");
  Duck duck = Duck();
  duck.fly();
  duck.swim();
  duck.stopSwim();

  print("\n=== Fish ===");
  Fish fish = Fish();
  fish.swim();

  print("\n=== Airplane ===");
  Airplane airplane = Airplane();
  airplane.fly();
}

11.Sealed Class & Pattern Matching

Pembahasan Implementasi Sealed Class dengan Pattern Matching pada Dart:

  • Pada program, sealed class Status digunakan untuk merepresentasikan beberapa kondisi, yaitu Loading, Loaded, dan Failed.
  • Setiap class turunan memiliki peran yang berbeda, seperti Loaded yang menyimpan data dan Failed yang menyimpan pesan error.
  • Struktur ini memastikan semua kemungkinan kondisi sudah didefinisikan dalam satu tempat sehingga lebih aman dan terkontrol.
  • Pada fungsi handleStatus(), digunakan pattern matching dalam switch-case untuk mengecek tipe objek secara langsung.
  • Setiap kondisi akan diproses sesuai jenisnya, seperti menampilkan "Loading...", data, atau pesan error.
  • Dengan pendekatan ini, penulisan kode menjadi lebih ringkas, jelas, dan mudah dipahami.
  • Hal ini menunjukkan bahwa kombinasi sealed class dan pattern matching membantu pengelolaan kondisi program secara lebih terstruktur.

Kode Dart:


// Nama  : Yoza Ulpia Rafila
// NIM   : 251410004
// Kelas : SI2A

// Sealed class
sealed class Status {
  const Status();
}

class Loading extends Status {
  const Loading();
}

class Loaded extends Status {
  final String data;
  const Loaded(this.data);
}

class Failed extends Status {
  final String error;
  const Failed(this.error);
}

// Function handle status
void handleStatus(Status status) {
  switch (status) {
    case Loading():
      print("Loading...");
      return;

    case Loaded(:final data):
      print("Data: $data");
      return;

    case Failed(:final error):
      print("Error: $error");
      return;
  }
}

void main() {
  handleStatus(const Loading());
  handleStatus(const Loaded("Data berhasil"));
  handleStatus(const Failed("Terjadi error"));
}

12.State Management dengan Sealed Class

Pembahasan Studi Kasus State Management Menggunakan Sealed Class pada Dart:

  • Pada program, state management diimplementasikan menggunakan sealed class AppState untuk merepresentasikan berbagai kondisi aplikasi.
  • Class AppState memiliki beberapa turunan yaitu LoadingState, SuccessState, dan ErrorState yang mewakili kondisi berbeda.
  • Setiap state memiliki fungsi masing-masing, seperti SuccessState yang menyimpan data dan ErrorState yang menyimpan pesan kesalahan.
  • Dengan struktur sealed class, seluruh kemungkinan state sudah ditentukan dalam satu file sehingga lebih aman dan terkontrol.
  • Pada fungsi updateUI(), digunakan pattern matching dalam switch-case untuk menangani setiap state secara spesifik.
  • Setiap kondisi akan memanggil fungsi yang sesuai, seperti showLoading(), showItems(), dan showError().
  • Dengan pendekatan ini, alur program menjadi lebih jelas, terstruktur, dan mudah dikembangkan.

Kode Dart:


// Nama  : Yoza Ulpia Rafila
// NIM   : 251410004
// Kelas : SI2A

// Sealed class
sealed class AppState {
  const AppState();
}

// State Loading
class LoadingState extends AppState {
  const LoadingState();
}

// State Success
class SuccessState extends AppState {
  final List items;

  const SuccessState(this.items);
}

// State Error
class ErrorState extends AppState {
  final String message;

  const ErrorState(this.message);
}

// Handler (Pattern Matching)
void updateUI(AppState state) {
  switch (state) {
    case LoadingState():
      showLoading();
      break;

    case SuccessState(:final items):
      showItems(items);
      break;

    case ErrorState(:final message):
      showError(message);
      break;
  }
}

// UI Functions
void showLoading() {
  print("⏳ Loading...");
}

void showItems(List items) {
  print("✅ Success! Items:");
  for (int i = 0; i < items.length; i++) {
    print("${i + 1}. ${items[i]}");
  }
}

void showError(String message) {
  print("❌ Error: $message");
}

// Main Program
void main() {
  updateUI(const LoadingState());

  updateUI(const SuccessState([
    "Item A",
    "Item B",
    "Item C"
  ]));

  updateUI(const ErrorState("Gagal memuat data"));
}
